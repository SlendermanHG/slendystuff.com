QwertyLock Privacy and Security

QwertyLock is designed for short private text conversations where a shared phrase is exchanged separately from the message link.

Back to QwertyLock

What the tool stores

  • The server stores encrypted QLR message blocks and random message IDs until the selected open limit or selected expiry is reached.
  • The server does not need the shared phrase, plaintext message, account, phone number, or contact list to deliver a message.
  • No cookies are used by the QwertyLock page. The browser may keep ordinary visit history unless the user clears it or uses private browsing.
  • The browser uses local storage only to remember hashes of messages already decoded on that same browser, so it can warn against reopening them.
  • The browser uses session storage for a temporary decode lease while opening a linked message.

What stays in the browser

  • Plaintext and the shared phrase are handled in the page by the browser's Web Crypto API.
  • The shared phrase is never intentionally sent to the QwertyLock server.
  • The optional phrase QR is generated on the page and contains the shared phrase as plain text for in-person exchange only.
  • Text-to-speech and speech-to-text features depend on the browser and device. They may use device or cloud speech services controlled by the operating system or browser vendor.

Security model

  • Messages use AES-GCM encryption with a fresh random salt and nonce per encrypted block.
  • The encryption key is derived from the shared phrase with PBKDF2-SHA-256.
  • Open limit choices are one successful decode or two successful decodes. After the final allowed decode, the server deletes the encrypted block.
  • Expiration choices are 10 minutes, 1 hour, 24 hours, or Forever. Forever means no timer expiry; the open limit still removes the encrypted block.

Limits

  • Do not use QwertyLock as a guarantee against compromised phones, malware, screenshots, screen recording, shoulder surfing, hostile keyboards, or someone with access to your unlocked device.
  • Anyone with both the message link and shared phrase can decode the message if the selected open limit and expiry have not been reached.
  • Do not text the shared phrase in the same SMS/MMS thread as encrypted links.
  • Strong public security claims should wait for an independent cryptography/security review. No formal outside audit is claimed here.

Safe use

  • Start by exchanging the shared phrase face-to-face, by voice, or over a separate trusted channel.
  • For in-person setup, use Phrase QR only while both people are physically present.
  • Send only QwertyLock links by SMS/MMS.
  • Use 10 minutes for sensitive messages, 1 hour for normal back-and-forth, 24 hours only when the receiver may not respond quickly, and Forever only when you accept that encrypted text may remain on the server until the open limit is reached.
  • Clear the page after reading or sending.
